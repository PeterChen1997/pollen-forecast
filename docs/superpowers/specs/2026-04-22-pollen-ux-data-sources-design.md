# 花粉雷达：数据来源展示、城市切换与附近城市查询设计

**日期：** 2026-04-22

## 目标

本轮改造聚焦 4 件事：

1. 在界面内明确展示当前花粉数据来源。
2. 支持手动切换城市，并按该城市查看附近已监测城市的花粉情况。
3. 修复当前界面的关键问题，提升本地开发与线上展示一致性。
4. 为后续扩展更多 pollen forecast 数据源保留清晰结构。

## 现状问题

### 1. 数据来源展示不完整

- 前端已拿到 `source` 字段，但仅在城市列表的少数 fallback 城市上显示标签。
- `weatherdt` 作为主源没有单独标签。
- 城市详情弹窗、地图弹窗、定位城市 banner 未展示来源。
- 页脚文案写死为 `graph.weatherdt.com`，与实际多源回退不一致。

### 2. 缺少城市切换能力

- 当前“附近城市”只依赖浏览器定位。
- 用户不能手动输入某个城市，查看该城市附近的花粉情况。
- 已有后端 `expandedCityMap` 仅用于反向地理编码，没有对前端开放为搜索数据。

### 3. 代码边界不清晰

- `frontend/src/App.tsx` 集中了数据请求、定位、距离排序、UI 组装。
- 前后端都缺少统一的“城市参考点 / 数据来源元信息”抽象。

### 4. 本地开发体验不正确

- Vite 没有 `/api` 代理，前端 dev server 直接请求自身，导致 JSON parse error。

## 根因分析

1. **来源标签缺失**：前端只定义了 `qweather` / `nearby` 标签，没有完整的 source metadata。
2. **附近排序耦合定位**：距离计算直接绑定 `userLocation`，没有“当前参考城市”的独立状态。
3. **城市搜索数据未暴露**：后端没有提供 major + expanded 城市目录给前端搜索使用。
4. **开发代理缺失**：Vite 默认端口下 `/api/*` 未转发到 Bun 后端。

## 设计方案

### A. 统一数据来源元信息

新增前端来源元信息模块，统一维护：

- `weatherdt`
- `qweather`
- `nearby`

每个来源包含：

- 短标签
- 完整名称
- 说明文案
- 是否推算数据

展示位置：

- 城市列表
- 我的城市 banner
- 地图弹窗
- 城市详情弹窗
- 页脚“今日数据源汇总”

### B. 引入“当前参考城市”状态

前端新增独立状态 `referenceCity`：

- 默认优先使用定位城市。
- 用户手动选择城市后，改为手动参考城市。
- 附近城市列表、距离文案、地图聚焦均围绕该参考城市展开。

这使“距您”与“距某城市”两种模式可以共存：

- 定位模式：显示 `距您`
- 手动切换模式：显示 `距<城市名>`

### C. 城市切换能力

后端新增城市目录接口，暴露：

- 已监测 major cities
- 扩展搜索城市（来自 `expandedCityMap`）

前端新增城市切换组件：

- 支持输入关键词筛选城市
- 支持切换到未单独监测的扩展城市
- 切换到扩展城市时，展示“该城市附近已监测城市”

### D. 地图与详情联动增强

地图新增 `focusCity`：

- 手动切换城市后，地图自动聚焦到参考城市。
- 若参考城市不在已监测列表中，也显示一个“参考城市”标记。

详情弹窗新增：

- 数据来源
- 来源说明
- 是否为推算/回退数据

### E. 代码结构优化

前端拆分：

- `frontend/src/lib/pollenSources.ts`：来源元信息
- `frontend/src/lib/cityReference.ts`：参考城市、距离、排序、筛选逻辑
- `frontend/src/components/CitySwitcher.tsx`：城市切换 UI

后端拆分：

- `backend/src/cityDirectory.ts`：城市目录、扩展城市、查找函数、搜索目录接口复用逻辑

### F. 开发环境修复

Vite 增加 `/api` 代理到 `http://localhost:8080`。

## 数据源策略

当前策略保持：

1. `weatherdt` 主源
2. `qweather` fallback
3. `nearby` 邻近城市推算

界面需明确告诉用户“当前展示的数据是哪个来源”，而不是假定只有单一来源。

## 暂不纳入本轮

- 接入新的商业级国外 pollen provider
- 做多数据源融合评分
- 做后台可配置 provider 权重
- 做账号体系 / 收藏城市

## 验证方案

### 自动化验证

- 为前端纯逻辑添加测试：来源元信息、参考城市排序、城市搜索过滤。
- 为后端纯逻辑添加测试：城市目录搜索、中文城市名匹配、最近 major city 查找。

### 手工验证

1. 首页列表能看到所有来源标签。
2. 详情弹窗能看到来源说明。
3. 手动切换到 major city 后，附近城市按该城市排序。
4. 手动切换到扩展城市后，仍可查看附近已监测城市。
5. Vite dev 模式下不再出现 `/api` JSON parse error。
